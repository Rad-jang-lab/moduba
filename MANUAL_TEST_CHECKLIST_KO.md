# Moduba 1차 리팩터링 검증용 수동 테스트 체크리스트 (Windows)

사전 준비(공통)
- 실행: `py dicom_viewer.py`
- 준비 파일
  - 단일 프레임 DICOM 1개
  - 멀티프레임 DICOM 1개
  - 정상 DICOM 여러 개가 있는 폴더
  - 압축 DICOM(디코더 없음 환경에서 실패 재현용) 1개
  - 비DICOM 파일(.txt 등) 1개

## 1) 단일 DICOM 열기
- [ ] `DICOM 열기` 클릭 → 정상 DICOM 1개 선택
- [ ] 이미지가 캔버스에 표시된다
- [ ] 상단에 `이미지: 1 / 1`, `프레임: 1 / 1`이 표시된다
- [ ] 환자/모달리티 요약 텍스트가 갱신된다

## 2) 폴더 열기
- [ ] `폴더 열기` 클릭 → 정상 DICOM + 비DICOM이 섞인 폴더 선택
- [ ] 첫 번째 정상 DICOM이 바로 표시된다
- [ ] 결과 팝업에 등록 개수/제외 개수가 보인다
- [ ] `이전 이미지/다음 이미지` 버튼으로 파일 이동이 된다

## 3) 프레임 이동(멀티프레임)
- [ ] 멀티프레임 DICOM 열기
- [ ] `다음 프레임`/`이전 프레임` 버튼으로 프레임 번호가 변한다
- [ ] `Shift+PgDn`/`Shift+PgUp` 단축키도 동일하게 동작한다
- [ ] 첫/마지막 프레임 경계에서 더 이동해도 크래시가 없다

## 4) 비교/썸네일(멀티뷰) 동작
- [ ] 폴더 로드 후 `단일/멀티 전환`으로 멀티뷰 진입
- [ ] 썸네일 그리드가 보이고, 타일 선택 시 상태가 바뀐다
- [ ] Enter 또는 더블클릭으로 단일 보기 진입 시 해당 이미지가 열린다
- [ ] `Compare Mode` 체크 후 좌/우 각각 `폴더 열기`로 로드
- [ ] 좌/우 `이전/다음 이미지`, `이전/다음 프레임`이 정상 동작한다
- [ ] `Sync` On 시 한쪽 이동에 반대쪽 인덱스가 동기화된다

## 5) 압축 DICOM 오류 처리
- [ ] 디코더 미설치 환경에서 압축 DICOM 열기 시도
- [ ] 오류 팝업에 디코더 설치 안내(`pylibjpeg`, `gdcm` 등)가 표시된다
- [ ] 앱이 종료되지 않고 계속 조작 가능하다

## 6) 폴더 진단
- [ ] `폴더 진단` 실행(정상/비DICOM/압축 파일 혼합 폴더)
- [ ] 진단 창에서 카테고리별 개수(일반 DICOM, 비DICOM, 표시 실패 등)가 표시된다
- [ ] 상세 목록에 파일명과 사유가 보인다
- [ ] `결과 저장`으로 txt 저장 후 파일이 생성되고 내용이 보존된다

## 빠른 합격 기준
- [ ] 앱 크래시 없음
- [ ] 단일/멀티/비교 모드 전환 시 화면 멈춤 없음
- [ ] 로딩 실패 케이스에서 사용자 메시지가 나오고 다음 동작 가능
